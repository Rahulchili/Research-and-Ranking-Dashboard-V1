0. How to Use This Playbook
This document is the operating manual for a Claude Cowork session (or Claude Code session) that will implement the v2.0 confidence-scoring refinement in Rahul's dashboard codebase. Load it as the working brief at the start of the session, alongside the companion PRD.
0.1 What is Different in v2.1
This revision (v2.1) of the implementation playbook fixes scope and correctness issues found while reviewing the v1.0 playbook against the v2.0 PRD. Specifically:
Language-agnostic. The PRD says the implementation surface is a local HTML/JS dashboard. The v1.0 playbook assumed a Python backend without checking. v2.1 requires Pass 1 to confirm the actual stack first, and provides parallel reference implementations for both Python and JavaScript.
Dashboard contract preserved. v1.0 silently renamed PRD-mandated JSON field names (e.g. delivery_verdict → verdict) and dropped count fields the dashboard reads. v2.1 enforces an output-shape contract that matches PRD Section 10.2 / 10.3 / 10.4 exactly.
Direction-of-favorable fixed. v1.0 applied the tolerance band symmetrically for all categories. For revenue/EPS/margin/FCF, above-range is favorable; for capex/opex/tax-rate, below-range is favorable. v2.1 makes this explicit per category.
Recency weighting default aligned with PRD. v1.0 defaulted use_recency_weighting=True; PRD §11.1 specifies default false. v2.1 matches the PRD.
Pair-ID format normalized to the PRD example (Q1_TO_Q2) — not the double-underscore form.
Open-ended guidance (e.g. "at least $10B") now has explicit handling in the scoring spec, per PRD §9.
Adapter responsibilities are spelled out so the dashboard JSON consumer keeps working through cutover.
Patch-level changes in v2.1.1 (full PRD coverage audit against the v2.0 PRD). These are corrections to v2.1, not new functionality:
Tolerance band honored in discrete scoring. PRD §6.1.1 and Glossary §4 define in-line as "within the guided range, or within the tolerance band around a point estimate." v2.1's discrete_verdict ignored the tolerance band on point estimates, which made the discrete and continuous scores disagree on point-estimate cases just below the point. Fixed in both Python and TypeScript reference implementations.
Test expected-values corrected. Pass 3 cases S3, S6, S12, and S16 had expected continuous-score numbers that did not match the PRD §6.1.2 formula (the formula uses midpoint as the denominator; earlier expected values appeared to use various other denominators). Recomputed end-to-end: S3 ≈ 0.71, S6 = 0.90, S12 = 0.60, S16 = 0.70 with the arithmetic shown inline.
Three missing edge-case flags wired into the pipeline. v2.1 defined PRE_ANNOUNCEMENT_USED, REVISED_GUIDANCE_DETECTED, and FISCAL_CALENDAR_CHANGE as DataQualityFlag values but never set them anywhere — test T13 in particular would have failed. v2.1.1 adds carrier fields on QuarterRecord (actuals_source_kind, fiscal_calendar_changed) for the adapter to populate, and emits each flag in the pipeline based on the carried signal. Revised guidance is detected via a non-empty ClaimRecord.revisions list.
Forbidden-copy anti-grep extended to cover the rest of PRD §13 (100% confidence, Latest guidance delivered/missed, 8 verified quarters, Based on 10 quarters), not only the 10 quarters family.
PRD §10.2 flat-field claim schema honored on output. v2.1 modeled the guidance bounds as a nested GuidanceRange object on ClaimRecord, which gave clean validation but emitted claim.guidance.low instead of the flat claim.guidance_low / guidance_high / guidance_midpoint / guidance_unit fields PRD §10.2 documents. The adapter's result_to_dashboard_json now flattens these on the way out so the dashboard reads exactly the PRD shape.

2. Operating Principles for Claude Cowork
2.1 Code Behavior Rules
Discovery before code. No file is created or modified before Pass 1 completes.
Read before write. Every file Cowork modifies must first be read in full; the reasoning for the change is quoted in the commit message or a code comment.
Tests-first within each pass. Each pass writes its test fixtures and test signatures before the implementation, then makes them pass.
Branch per pass. Each pass is a separate git branch named feature/v2-confidence-pass-N, merged into a v2-confidence integration branch after its gate clears.
Run gates after every edit. Test runner and linter after every meaningful save; strict type checker at the end of each pass.
No silent type coercion. Pydantic v2 models use strict=True; TypeScript uses "strict": true.
Output contract is sacred. The JSON shape consumed by dashboard/ may gain fields but never lose them or have them renamed without an explicit ADR.
Surface ambiguity. If a spec in this playbook conflicts with the PRD, surface and ask. The PRD is the semantic authority; this playbook is the implementation guide.
2.2 Allowed Dependencies
Use only these libraries. Do not introduce others without explicit human approval.
Python path
**Why numpy and not statistics:** either works. Use numpy only if it is already imported elsewhere in the repo. If not, use `statistics.stdev` and `math.sqrt` and note the substitution in the discovery report.
JavaScript path
**Why zod and not plain TypeScript interfaces:** the dashboard consumes JSON. We need runtime validation at the package boundary, not just compile-time types. If the repo already standardizes on a different validator (yup, valibot, io-ts), use that and document the substitution.
2.3 Style & Type Conventions (Python)
Python 3.11+ syntax; use match statements and exception groups where natural.
Full type annotations on every public function and method.
Use Pydantic v2 BaseModel for all data records; do not mix in dataclasses.
StrEnum subclasses for fixed-vocabulary fields (status, verdict, category).
Docstrings: Google style, with an Invariants section listing what the function guarantees.
No mutable default arguments. Ever.
ruff format and ruff check --fix run before every commit.
2.4 Style & Type Conventions (JavaScript)
TypeScript with "strict": true; no implicit any.
ES modules; no CommonJS in new code.
Zod schemas as the source of truth; types derived via z.infer.
Pure functions for scoring math; no class instances on the hot path.
Use number | null for missing values, never undefined, because the JSON contract requires explicit nulls.
ESLint + Prettier before every commit.

from typing import Annotated, Optional

    ConfigDict,

SCHEMA_VERSION = "2.0"
 
 
class ClaimCategory(StrEnum):
    REVENUE = "revenue"
    EPS = "eps"

    OPERATING_MARGIN = "operating_margin"

    """Which side of the guided range is favorable for this metric.
 
    Revenue beat = above range (UP_IS_GOOD).
    Capex under-spend = below range (DOWN_IS_GOOD).
    Tax rate below guide = below range (DOWN_IS_GOOD).
    """
    UP_IS_GOOD = "up_is_good"
    DOWN_IS_GOOD = "down_is_good"
 
 
# Mapping is authoritative; do not duplicate in scoring.py
DIRECTION_BY_CATEGORY: dict[ClaimCategory, ClaimDirection] = {
    ClaimCategory.REVENUE: ClaimDirection.UP_IS_GOOD,
    ClaimCategory.EPS: ClaimDirection.UP_IS_GOOD,
    ClaimCategory.GROSS_MARGIN: ClaimDirection.UP_IS_GOOD,
    ClaimCategory.OPERATING_MARGIN: ClaimDirection.UP_IS_GOOD,
    ClaimCategory.FREE_CASH_FLOW: ClaimDirection.UP_IS_GOOD,
    ClaimCategory.SEGMENT_REVENUE: ClaimDirection.UP_IS_GOOD,
    ClaimCategory.PRODUCT_LAUNCH: ClaimDirection.UP_IS_GOOD,
    ClaimCategory.CUSTOMER_MILESTONE: ClaimDirection.UP_IS_GOOD,
    ClaimCategory.CAPEX: ClaimDirection.DOWN_IS_GOOD,
    ClaimCategory.OPERATING_EXPENSE: ClaimDirection.DOWN_IS_GOOD,
    ClaimCategory.TAX_RATE: ClaimDirection.DOWN_IS_GOOD,
    ClaimCategory.OTHER: ClaimDirection.UP_IS_GOOD,
}
 
 
class DeliveryVerdict(StrEnum):
    DELIVERED = "delivered"   # in-range or beat in the favorable direction (capped at 1.00)
    PARTIAL = "partial"       # 0%–5% miss in the unfavorable direction
    MISSED = "missed"         # > 5% miss in the unfavorable direction
    WITHDRAWN = "withdrawn"   # rescinded before target quarter reported
    PENDING = "pending"       # target quarter not yet reported
 
 
class PairStatus(StrEnum):
    CLOSED = "closed"
    PENDING = "pending"
    INSUFFICIENT = "insufficient"
 
 
class DataQualityFlag(StrEnum):
    MISSING_TRANSCRIPT = "missing_transcript"
    MISSING_ACTUALS = "missing_actuals"
    INSUFFICIENT_WINDOW = "insufficient_window"
    NO_GUIDANCE_ISSUED = "no_guidance_issued"
    REVISED_GUIDANCE_DETECTED = "revised_guidance_detected"
    PRE_ANNOUNCEMENT_USED = "pre_announcement_used"
    FISCAL_CALENDAR_CHANGE = "fiscal_calendar_change"
 
 
class GuidanceRange(BaseModel):
    """Guided range or point estimate.
 
    Open-ended guidance ("at least $10B") is represented as
    `high=None` — the scoring code treats it as one-sided.
    """
    model_config = ConfigDict(strict=True, extra="forbid")
 
    low: float
    high: float | None = None        # None for one-sided open-ended
    midpoint: float | None = None
    unit: str = ""
 
    @model_validator(mode="after")
    def _fill_and_validate(self) -> "GuidanceRange":
        if self.high is not None and self.low > self.high:
            raise ValueError(f"low ({self.low}) must be <= high ({self.high})")
        if self.midpoint is None:
            if self.high is None:
                # Open-ended: midpoint defaults to the low end for sandbag math
                self.midpoint = self.low
            else:
                self.midpoint = (self.low + self.high) / 2.0
        return self
 
 
class ClaimRecord(BaseModel):
    """A single measurable forward-looking management claim.
 
    Serialization aliases match PRD §10.2 exactly so the dashboard
    JSON consumer sees the field names it expects.
    """
    model_config = ConfigDict(
        strict=True, extra="forbid", populate_by_name=True,
    )
 
    claim_id: str
    ticker: str
    guidance_quarter: str
    target_quarter: str
    category: ClaimCategory
    weight: NonNegativeFloat
    guidance: GuidanceRange
    tolerance_pct: NonNegativeFloat = 0.02
    evidence_quote: str
    evidence_source: str
    evidence_speaker: str = ""
    actual: float | None = None
    actual_source: str = ""
 
    # PRD-mandated field name is "delivery_verdict"; we expose it
    # under that name in JSON. The Python attribute is kept short.
    verdict: DeliveryVerdict = Field(
        default=DeliveryVerdict.PENDING, alias="delivery_verdict",
    )
    delivery_score_discrete: float | None = None
    delivery_score_continuous: float | None = None
    signed_beat_pct: float | None = None
    revisions: list[dict] = Field(default_factory=list)
    withdrawn_reason: str = ""
    computed_at: datetime | None = None
 
    @field_validator("delivery_score_discrete", "delivery_score_continuous")
    @classmethod
    def _bounded(cls, v: float | None) -> float | None:
        if v is None:
            return v
        if not 0.0 <= v <= 1.0:
            raise ValueError(f"score must be in [0, 1], got {v}")
        return v
 
 
class QuarterRecord(BaseModel):
    """One fiscal quarter of data for one ticker.
 
    The actuals_source_kind and fiscal_calendar_changed fields are
    populated by the adapter from the legacy data. The pipeline reads
    them to emit the matching DataQualityFlag values per PRD §9.
    """
    model_config = ConfigDict(strict=True, extra="forbid")
 
    ticker: str
    fiscal_label: str
    calendar_period_end: datetime | None = None
    transcript_available: bool = True
    actuals_available: bool = True
    forward_claims: list[ClaimRecord] = Field(default_factory=list)
    actuals_by_category: dict[str, float] = Field(default_factory=dict)
 
    # PRD §9 — pre-announcement: when the company released numbers
    # before the official 10-Q. The adapter sets this; the pipeline
    # emits PRE_ANNOUNCEMENT_USED when any quarter uses it.
    actuals_source_kind: str = "official"  # "official" | "pre_announcement"
 
    # PRD §9 — fiscal calendar change: if this quarter is the first
    # under a new fiscal calendar, the adapter sets this True and
    # the pipeline emits FISCAL_CALENDAR_CHANGE.
    fiscal_calendar_changed: bool = False
 
 
class PairRecord(BaseModel):
    """One guidance-to-actual pair.
 
    Field names match PRD §10.3 exactly. Per-verdict count fields
    are populated so the dashboard audit table can render without
    additional lookups.
    """
    model_config = ConfigDict(strict=True, extra="forbid")
 
    pair_id: str
    guidance_quarter: str
    target_quarter: str
    status: PairStatus
    claims: list[ClaimRecord] = Field(default_factory=list)
    claim_count: NonNegativeInt = 0
    scorable_claim_count: NonNegativeInt = 0
    delivered_claims: NonNegativeInt = 0     # PRD §10.3
    partial_claims: NonNegativeInt = 0       # PRD §10.3
    missed_claims: NonNegativeInt = 0        # PRD §10.3
    withdrawn_claims: NonNegativeInt = 0     # PRD §10.3
    pair_score: float | None = None
    pair_score_continuous: float | None = None
    summary: str = ""
 
    @field_validator("pair_score", "pair_score_continuous")
    @classmethod
    def _bounded_pair(cls, v: float | None) -> float | None:
        if v is None:
            return v
        if not 0.0 <= v <= 1.0:
            raise ValueError(f"pair score must be in [0, 1], got {v}")
        return v

from copy import deepcopy
 
from confidence.models import (
    ClaimDirection,

    DIRECTION_BY_CATEGORY,

EPS = 1e-9
 
 
def _direction(claim: ClaimRecord) -> ClaimDirection:
    return DIRECTION_BY_CATEGORY.get(claim.category, ClaimDirection.UP_IS_GOOD)
 
 
def continuous_score(claim: ClaimRecord) -> float | None:
    """Magnitude-aware delivery score in [0, 1].
 
    PRD §6.1.2 with direction-of-favorable correction:
        - UP_IS_GOOD:   above range = full credit; below range = shortfall.
        - DOWN_IS_GOOD: below range = full credit; above range = overspend.
    Open-ended guidance (high=None) is treated as one-sided around `low`.
 
    Returns None if claim is pending; 0.25 if withdrawn.
 
    Invariants:
        - return value is None, 0.25, or in [0.0, 1.0]
        - never raises for valid input
    """
    if claim.actual is None:
        return None
    if claim.verdict == DeliveryVerdict.WITHDRAWN:
        return 0.25
 
    g = claim.guidance
    direction = _direction(claim)
    a = claim.actual
    midpoint = g.midpoint if g.midpoint is not None else g.low
    denom = max(abs(midpoint), EPS)
 
    # Tolerance applies only to point guidance (low == high or high is None)
    is_point = (g.high is None) or (g.low == g.high)
    t_abs = claim.tolerance_pct * denom if is_point else 0.0
 
    if direction == ClaimDirection.UP_IS_GOOD:
        effective_low = g.low - t_abs
        # For UP_IS_GOOD, beats above high are still full credit
        if a >= effective_low:
            return 1.0
        shortfall_pct = (effective_low - a) / denom
        raw = 1.0 - 10.0 * shortfall_pct
        return max(0.0, min(1.0, raw))
 
    # DOWN_IS_GOOD: a below the range high is good; above is bad
    if g.high is None:
        # Open-ended "at most X": treat low as the ceiling
        effective_high = g.low + t_abs
    else:
        effective_high = g.high + t_abs
    if a <= effective_high:
        return 1.0
    overshoot_pct = (a - effective_high) / denom
    raw = 1.0 - 10.0 * overshoot_pct
    return max(0.0, min(1.0, raw))
 
 
def discrete_verdict(claim: ClaimRecord) -> DeliveryVerdict:
    """v1.0-compatible discrete verdict, direction-aware.
 
    Per PRD §6.1.1 and Glossary §4, an actual that falls *within the
    tolerance band around a point estimate* is in-line / delivered. The
    tolerance band is applied here so the discrete and continuous scores
    agree on point-estimate cases.
 
    Buckets:
        delivered:  actual is on the favorable side of the range,
                    or within the tolerance band around a point estimate
        partial:    miss is 0-5% beyond the tolerance band (unfavorable)
        missed:     miss is > 5% beyond the tolerance band (unfavorable)
        withdrawn:  preserved if already set
        pending:    actual is None
    """
    if claim.actual is None:
        return DeliveryVerdict.PENDING
    if claim.verdict == DeliveryVerdict.WITHDRAWN:
        return DeliveryVerdict.WITHDRAWN
 
    g = claim.guidance
    direction = _direction(claim)
    a = claim.actual
    midpoint = g.midpoint if g.midpoint is not None else g.low
    denom = max(abs(midpoint), EPS)
 
    # Tolerance applies only to point guidance (low == high or open-ended).
    is_point = (g.high is None) or (g.low == g.high)
    t_abs = claim.tolerance_pct * denom if is_point else 0.0
 
    if direction == ClaimDirection.UP_IS_GOOD:
        effective_low = g.low - t_abs
        if a >= effective_low:
            return DeliveryVerdict.DELIVERED
        shortfall_pct = (effective_low - a) / denom
        if shortfall_pct <= 0.05:
            return DeliveryVerdict.PARTIAL
        return DeliveryVerdict.MISSED
 
    # DOWN_IS_GOOD
    high_bound = g.high if g.high is not None else g.low
    effective_high = high_bound + t_abs
    if a <= effective_high:
        return DeliveryVerdict.DELIVERED
    overshoot_pct = (a - effective_high) / denom
    if overshoot_pct <= 0.05:
        return DeliveryVerdict.PARTIAL
    return DeliveryVerdict.MISSED
 
 
def discrete_score(verdict: DeliveryVerdict) -> float | None:
    """v1.0-compatible discrete score."""
    return {
        DeliveryVerdict.DELIVERED: 1.00,
        DeliveryVerdict.PARTIAL: 0.50,
        DeliveryVerdict.MISSED: 0.00,
        DeliveryVerdict.WITHDRAWN: 0.25,
        DeliveryVerdict.PENDING: None,
    }[verdict]
 
 
def signed_beat_pct(claim: ClaimRecord) -> float | None:
    """(actual - midpoint) / |midpoint|. Raw sign, not direction-adjusted.
 
    None when pending. The sandbag module is responsible for flipping
    the sign on DOWN_IS_GOOD metrics if it ever needs to combine them
    (currently sandbag only inspects revenue and EPS, both UP_IS_GOOD).
    """
    if claim.actual is None:
        return None
    g = claim.guidance
    midpoint = g.midpoint if g.midpoint is not None else g.low
    denom = max(abs(midpoint), EPS)
    return (claim.actual - midpoint) / denom
 
 
def score_claim(claim: ClaimRecord, config: ConfidenceConfig) -> ClaimRecord:
    """Pure function: return a scored copy of `claim`. Never mutates input."""
    out = deepcopy(claim)
    verdict = discrete_verdict(out)
    out.verdict = verdict
    out.delivery_score_discrete = discrete_score(verdict)
    out.delivery_score_continuous = continuous_score(out)
    out.signed_beat_pct = signed_beat_pct(out)
    return out

// src/confidence/scoring.ts
import {
  ClaimRecord, ConfidenceConfig, DIRECTION_BY_CATEGORY,
  type DeliveryVerdict,
} from "./models";
 
const EPS = 1e-9;
 
function direction(c: ClaimRecord) {
  return DIRECTION_BY_CATEGORY[c.category] ?? "up_is_good";
}
 
export function continuousScore(c: ClaimRecord): number | null {
  if (c.actual === null) return null;
  if (c.delivery_verdict === "withdrawn") return 0.25;
 
  const g = c.guidance;
  const a = c.actual;
  const midpoint = g.midpoint ?? g.low;
  const denom = Math.max(Math.abs(midpoint), EPS);
 
  const isPoint = g.high === null || g.low === g.high;
  const tAbs = isPoint ? c.tolerance_pct * denom : 0;
 
  if (direction(c) === "up_is_good") {
    const effectiveLow = g.low - tAbs;
    if (a >= effectiveLow) return 1.0;
    const shortfallPct = (effectiveLow - a) / denom;
    return Math.max(0, Math.min(1, 1 - 10 * shortfallPct));
  }
 
  // down_is_good
  const high = g.high ?? g.low;
  const effectiveHigh = high + tAbs;
  if (a <= effectiveHigh) return 1.0;
  const overshootPct = (a - effectiveHigh) / denom;
  return Math.max(0, Math.min(1, 1 - 10 * overshootPct));
}
 
export function discreteVerdict(c: ClaimRecord): DeliveryVerdict {
  if (c.actual === null) return "pending";
  if (c.delivery_verdict === "withdrawn") return "withdrawn";
 
  const g = c.guidance;
  const a = c.actual;
  const midpoint = g.midpoint ?? g.low;
  const denom = Math.max(Math.abs(midpoint), EPS);
 
  // PRD §6.1.1 / Glossary §4: an actual within the tolerance band
  // around a point estimate is in-line. Discrete and continuous
  // verdicts must agree on this case.
  const isPoint = g.high === null || g.low === g.high;
  const tAbs = isPoint ? c.tolerance_pct * denom : 0;
 
  if (direction(c) === "up_is_good") {
    const effectiveLow = g.low - tAbs;
    if (a >= effectiveLow) return "delivered";
    const shortfallPct = (effectiveLow - a) / denom;
    return shortfallPct <= 0.05 ? "partial" : "missed";
  }
  const high = g.high ?? g.low;
  const effectiveHigh = high + tAbs;
  if (a <= effectiveHigh) return "delivered";
  const overshootPct = (a - effectiveHigh) / denom;
  return overshootPct <= 0.05 ? "partial" : "missed";
}
 
export function discreteScore(v: DeliveryVerdict): number | null {
  return ({
    delivered: 1.0, partial: 0.5, missed: 0.0,
    withdrawn: 0.25, pending: null,
  } as const)[v];
}
 
export function signedBeatPct(c: ClaimRecord): number | null {
  if (c.actual === null) return null;
  const g = c.guidance;
  const midpoint = g.midpoint ?? g.low;
  const denom = Math.max(Math.abs(midpoint), EPS);
  return (c.actual - midpoint) / denom;
}
 
export function scoreClaim(c: ClaimRecord, _cfg: ConfidenceConfig): ClaimRecord {
  const out = structuredClone(c);
  out.delivery_verdict = discreteVerdict(out);
  out.delivery_score_discrete = discreteScore(out.delivery_verdict);
  out.delivery_score_continuous = continuousScore(out);
  out.signed_beat_pct = signedBeatPct(out);
  return out;
}

def _counts(claims: list[ClaimRecord]) -> dict[DeliveryVerdict, int]:
    c: Counter[DeliveryVerdict] = Counter()
    for cl in claims:
        c[cl.verdict] += 1
    return c
 
 
def build_pair(
    guidance_q: QuarterRecord,
    target_q: QuarterRecord | None,
    config: ConfidenceConfig,
) -> PairRecord:
    """Construct a PairRecord from a guidance quarter and its target."""
    pair_id = _pair_id(
        guidance_q.fiscal_label,
        target_q.fiscal_label if target_q else "next",
    )
    # Defensive copy — never mutate the caller's quarter records
    claims_in: list[ClaimRecord] = [c.model_copy() for c in guidance_q.forward_claims]
 
    # Pending: no target, or target has no actuals
    if target_q is None or not target_q.actuals_available:
        for c in claims_in:
            c.verdict = DeliveryVerdict.PENDING
            c.weight = weight_for(c.category)
        return PairRecord(
            pair_id=pair_id,
            guidance_quarter=guidance_q.fiscal_label,
            target_quarter=target_q.fiscal_label if target_q else "next",
            status=PairStatus.PENDING,
            claims=claims_in,
            claim_count=len(claims_in),
            scorable_claim_count=0,
            summary="Awaiting target quarter actuals.",
        )
 
    # Closed: score each claim against its category's actual
    scored: list[ClaimRecord] = []
    for c in claims_in:
        actual = target_q.actuals_by_category.get(c.category.value)
        c.actual = actual
        c.weight = weight_for(c.category)
        scored.append(score_claim(c, config))
 
    scorable = [
        s for s in scored
        if s.delivery_score_discrete is not None
        and s.verdict != DeliveryVerdict.WITHDRAWN
    ]
    averageable = [s for s in scored if s.delivery_score_discrete is not None]
    counts = _counts(scored)
 
    base_kwargs = dict(
        pair_id=pair_id,
        guidance_quarter=guidance_q.fiscal_label,
        target_quarter=target_q.fiscal_label,
        claims=scored,
        claim_count=len(scored),
        scorable_claim_count=len(scorable),
        delivered_claims=counts.get(DeliveryVerdict.DELIVERED, 0),
        partial_claims=counts.get(DeliveryVerdict.PARTIAL, 0),
        missed_claims=counts.get(DeliveryVerdict.MISSED, 0),
        withdrawn_claims=counts.get(DeliveryVerdict.WITHDRAWN, 0),
    )
 
    if len(scorable) < config.min_claims_per_pair:
        return PairRecord(
            **base_kwargs,
            status=PairStatus.INSUFFICIENT,
            summary=(
                f"Only {len(scorable)} scorable claim(s); "
                f"below threshold of {config.min_claims_per_pair}."
            ),
        )
 
    total_weight = sum(s.weight for s in averageable) or 1.0
    pair_score = sum(
        s.weight * s.delivery_score_discrete for s in averageable
    ) / total_weight
    pair_score_cont = sum(
        s.weight * (s.delivery_score_continuous or 0.0) for s in averageable
    ) / total_weight
 
    return PairRecord(
        **base_kwargs,
        status=PairStatus.CLOSED,
        pair_score=round(pair_score, 6),
        pair_score_continuous=round(pair_score_cont, 6),
        summary=_summarize(scored),
    )

# src/confidence/aggregation.py
from __future__ import annotations
 
import math
import statistics
from typing import Sequence
 
from confidence.models import ConfidenceConfig, PairRecord, PairStatus
 
 
def _scorable_pair_scores(pairs: Sequence[PairRecord]) -> list[float]:
    return [
        p.pair_score for p in pairs
        if p.status == PairStatus.CLOSED and p.pair_score is not None
    ]
 
 
def mcs(pairs: Sequence[PairRecord]) -> float | None:
    scores = _scorable_pair_scores(pairs)
    return sum(scores) / len(scores) if scores else None
 
 
def mcs_recency_weighted(
    pairs: Sequence[PairRecord], config: ConfidenceConfig,
) -> float | None:
    """Linear-decay weighted mean.
 
    PRD §6.4.1 specifies weights [0.6, 0.8, 1.0, 1.2, 1.4, 1.6, 1.8] for n=7.
    We generalize: linear from 0.6 (oldest) to 1.8 (newest) for any n >= 2.
    """
    scores = _scorable_pair_scores(pairs)
    if not scores:
        return None
    n = len(scores)
    if n == 1:
        return scores[0]
    weights = [0.6 + (1.8 - 0.6) * i / (n - 1) for i in range(n)]
    total_w = sum(weights)
    return sum(w * s for w, s in zip(weights, scores)) / total_w
 
 
def standard_error(pairs: Sequence[PairRecord]) -> float | None:
    scores = _scorable_pair_scores(pairs)
    if len(scores) < 2:
        return None
    sd = statistics.stdev(scores)
    return sd / math.sqrt(len(scores))
 
 
def ci95(pairs: Sequence[PairRecord]) -> tuple[float, float] | None:
    m = mcs(pairs)
    se = standard_error(pairs)
    if m is None or se is None:
        return None
    return (max(0.0, m - 1.96 * se), min(1.0, m + 1.96 * se))
 
 
def label_from_mcs(value: float | None, config: ConfidenceConfig) -> str:
    """Round to 1 decimal place, then bucket. Rounded-first prevents flicker."""
    if value is None:
        return "N/A"
    pct = round(value * 100, 1)
    if pct >= config.high_threshold * 100:
        return "High"
    if pct >= config.medium_threshold * 100:
        return "Medium"
    return "Low"
 
 
def label_sensitive(
    pairs: Sequence[PairRecord], config: ConfidenceConfig,
) -> bool:
    """True when the 95% CI spans two different labels."""
    interval = ci95(pairs)
    if interval is None:
        return False
    return label_from_mcs(interval[0], config) != label_from_mcs(interval[1], config)

def recent_trend(
    pairs: Sequence[PairRecord], config: ConfidenceConfig,
) -> str:
    """Improving | Stable | Deteriorating | Insufficient Data.
 
    Guard: requires >= recency_window scorable closed pairs.

    if len(scores) < config.recency_window:

    recent = scores[-config.recency_window:]
    recent_avg = sum(recent) / len(recent)
    overall = mcs(pairs)
    if overall is None:
        return "Insufficient Data"
 
    delta = recent_avg - overall
    if delta >= config.trend_delta:
        return "Improving"
    if delta <= -config.trend_delta:
        return "Deteriorating"
    return "Stable"

SANDBAG_CATEGORIES = {ClaimCategory.REVENUE, ClaimCategory.EPS}
 
 
def sandbag_index(pairs: Sequence[PairRecord]) -> float | None:
    """Mean signed_beat_pct across revenue and EPS claims in closed pairs."""
    beats: list[float] = []
    for p in pairs:
        if p.status != PairStatus.CLOSED:
            continue
        for c in p.claims:
            if c.category in SANDBAG_CATEGORIES and c.signed_beat_pct is not None:
                beats.append(c.signed_beat_pct)
    return sum(beats) / len(beats) if beats else None
 
 
def sandbag_label(value: float | None, config: ConfidenceConfig) -> str:
    if value is None:
        return ""
    if value >= config.sandbag_high_threshold:
        return "Likely sandbagging — chronic large beats"
    if value >= config.sandbag_mild_threshold:
        return "Mild conservatism"
    if value <= -config.sandbag_mild_threshold:
        return "Optimistic guidance — chronic shortfalls"
    return "Calibrated guidance"

# src/confidence/pipeline.py
from __future__ import annotations
 
from datetime import datetime, timezone
from typing import Sequence
 
from confidence.aggregation import (
    ci95, label_from_mcs, label_sensitive, mcs,
    mcs_recency_weighted, standard_error,
)
from confidence.models import (
    SCHEMA_VERSION,
    ConfidenceConfig, ConfidenceResult, DataQualityFlag,
    PairStatus, QuarterRecord,
)
from confidence.pair import build_all_pairs
from confidence.sandbag import sandbag_index, sandbag_label
from confidence.trend import recent_trend
 
 
def compute_confidence(
    quarters: Sequence[QuarterRecord],
    config: ConfidenceConfig | None = None,
    ticker: str | None = None,
) -> ConfidenceResult:
    """Top-level entry point for the v2.0 confidence pipeline.
 
    Args:
        quarters: chronological ascending list of QuarterRecord
        config: tunable thresholds; defaults if None
        ticker: optional override; otherwise read from quarters[0]
 
    Returns:
        ConfidenceResult — always; never raises for shape issues.
        Data-quality problems surface as data_quality_flags on the result.
    """
    cfg = config or ConfidenceConfig()
    flags: list[DataQualityFlag] = []
    tkr = ticker or (quarters[0].ticker if quarters else "UNKNOWN")
    today = datetime.now(timezone.utc).date().isoformat()
 
    # --- Shape & coverage flags --------------------------------------
    n = len(quarters)
    if n < 4:
        flags.append(DataQualityFlag.INSUFFICIENT_WINDOW)
    if quarters and not quarters[-1].transcript_available:
        flags.append(DataQualityFlag.MISSING_TRANSCRIPT)
    for q in quarters[:-1]:
        if not q.actuals_available:
            flags.append(DataQualityFlag.MISSING_ACTUALS)
            break
    if quarters and all(len(q.forward_claims) == 0 for q in quarters):
        flags.append(DataQualityFlag.NO_GUIDANCE_ISSUED)
 
    # PRD §9 — pre-announcement actuals were used somewhere in the window.
    if any(q.actuals_source_kind == "pre_announcement" for q in quarters):
        flags.append(DataQualityFlag.PRE_ANNOUNCEMENT_USED)
 
    # PRD §9 — fiscal calendar change occurred inside the window.
    if any(q.fiscal_calendar_changed for q in quarters):
        flags.append(DataQualityFlag.FISCAL_CALENDAR_CHANGE)
 
    # PRD §8.3 — revised guidance was detected on any claim. The
    # adapter populates ClaimRecord.revisions with the prior versions
    # and keeps the most recent revision as the scored claim.
    if any(
        bool(c.revisions)
        for q in quarters
        for c in q.forward_claims
    ):
        flags.append(DataQualityFlag.REVISED_GUIDANCE_DETECTED)
 
    # --- Build pairs --------------------------------------------------
    pairs = build_all_pairs(list(quarters), cfg)
    closed = [p for p in pairs if p.status == PairStatus.CLOSED]
    pending = [p for p in pairs if p.status == PairStatus.PENDING]
    insufficient = [p for p in pairs if p.status == PairStatus.INSUFFICIENT]
 
    # --- Aggregate ----------------------------------------------------
    m = mcs(pairs)
    if DataQualityFlag.NO_GUIDANCE_ISSUED in flags:
        label = "N/A"
    elif len(closed) >= cfg.min_closed_pairs_for_label:
        label = label_from_mcs(m, cfg)
    else:
        label = "N/A"
 
    se = standard_error(pairs)
    ci = ci95(pairs)
    sensitive = label_sensitive(pairs, cfg) if label != "N/A" else False
    m_rw = (
        mcs_recency_weighted(pairs, cfg)
        if cfg.use_recency_weighting else None
    )
    trend = recent_trend(pairs, cfg)
    sbi = sandbag_index(pairs)
    sbl = sandbag_label(sbi, cfg)
 
    return ConfidenceResult(
        schema_version=SCHEMA_VERSION,
        ticker=tkr,
        as_of_date=today,
        quarters_analyzed=n,
        closed_pairs_used=len(closed),
        pending_pairs_excluded=len(pending),
        insufficient_pairs_excluded=len(insufficient),
        mcs=round(m, 6) if m is not None else None,
        mcs_pct=round(m * 100, 1) if m is not None else None,
        mcs_recency_weighted=round(m_rw, 6) if m_rw is not None else None,
        mcs_recency_weighted_pct=round(m_rw * 100, 1) if m_rw is not None else None,
        standard_error=round(se, 6) if se is not None else None,
        ci95_low_pct=round(ci[0] * 100, 1) if ci else None,
        ci95_high_pct=round(ci[1] * 100, 1) if ci else None,
        confidence_level=label,
        label_sensitive=sensitive,
        recent_trend=trend,
        sandbag_index=round(sbi, 6) if sbi is not None else None,
        sandbag_label=sbl,
        method_note=(
            f"MCS computed from {len(closed)} closed guidance-to-actual "
            f"pairs across the last {n} quarters. Latest forward guidance "
            f"is pending and excluded."
        ),
        pairs=pairs,
        data_quality_flags=flags,
    )
10.3 Edge Cases Handled Here
This pipeline is where every PRD §9 edge case is enforced. Verify each one:
n < 4 → label='N/A', flag=INSUFFICIENT_WINDOW.
Latest transcript missing → flag emitted; closed pairs still scored.
Target actuals missing → pair is PENDING, excluded from score.
Zero or one measurable claim in a pair → pair is INSUFFICIENT, excluded.
No guidance issued at all → flag NO_GUIDANCE_ISSUED; label='N/A'.
Withdrawn claims score 0.25; counted in withdrawn_claims.
Single closed pair → MCS computed; SE=None; trend='Insufficient Data'; label='N/A' (below floor of 3).
Open-ended guidance ("at least $10B") handled inside continuous_score and discrete_verdict — pipeline does nothing special.
Capex/opex/tax-rate direction handled inside scoring — pipeline is direction-agnostic.
Pre-announcement used (PRD §9) → adapter sets QuarterRecord.actuals_source_kind = "pre_announcement" so the pipeline emits PRE_ANNOUNCEMENT_USED.
Revised guidance mid-quarter (PRD §8.3) → adapter places prior versions in ClaimRecord.revisions and the latest revision in the scored claim; the pipeline emits REVISED_GUIDANCE_DETECTED whenever any claim has a non-empty revisions list.
Fiscal calendar change (PRD §9) → adapter sets QuarterRecord.fiscal_calendar_changed = True on the first quarter under the new calendar; pipeline emits FISCAL_CALENDAR_CHANGE. Scoring continues on the new calendar — periods are never blended.
Multi-year guidance (PRD §9) → excluded at the adapter; never appears as a ClaimRecord in this pipeline.

Anti-Pattern | Why It's Rejected
Any hardcoded 7, 8, or 10 in the confidence module | The whole point of v2.0 is to derive these from quarters.length.
raise from inside compute_confidence() | Shape problems must surface as flags, not exceptions. Only programmer errors raise.
Mutable default arguments (def f(x=[])) | Standard Python footgun; ruff catches it.
Catching BaseException or bare except | Hides KeyboardInterrupt and SystemExit; never do this.
Adding pandas / polars / sklearn as new dependencies | Out of scope; statistics + numpy (or none) is sufficient.
Mutating a Pydantic / zod input model in place | All scoring functions take and return new instances.
UI label strings other than 'High' / 'Medium' / 'Low' / 'N/A' | These are exact PRD copy. Casing matters.
Renaming a JSON field the dashboard already reads | This breaks the dashboard. Add fields; never rename. Use the adapter to alias.
Tests that write to disk without using tmp_path / os.tmpdir() | Pollutes the repo and breaks parallel test runs.
Using float equality (a == b) on scores | Use math.isclose(a, b, abs_tol=1e-6) in tests.
Assuming capex/opex actual > guidance is a miss | Direction-of-favorable; under-spending capex is a delivery.

Pitfall | Defense
Forgetting to recompute mid-quarter when guidance is revised | ClaimRecord.revisions is a list; always score the most recent. Add a fixture in test_pipeline.py.
Confusing the two MCS variants | Display both fields when use_recency_weighting=True; the equal-weighted is the canonical headline number.
CI clipping at [0, 1] | ci95 must clip; don't let SE produce negative lower bound or > 1 upper bound.
Boundary rounding flicker | Always round to 1 decimal before comparing to thresholds, never compare on raw float.
